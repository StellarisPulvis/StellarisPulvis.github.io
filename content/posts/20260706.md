> 从零开始搭建开发环境、连接与控制

Raspberry Pi Pico 2W 是一块搭载 RP2350 芯片、自带 Wi-Fi 和 BLE 的微控制器开发板，可以用 MicroPython 轻松编程。本文将从固件烧录开始，带你一步步完成环境搭建、设备连接、文件传输到实际项目开发。

---

## 一、烧录 MicroPython 固件

刚买回来的 Pico 2W 是空板，需要先烧录 MicroPython 固件才能使用。

### 1. 下载固件

前往 [MicroPython 官方下载页面](https://micropython.org/download/RPI_PICO2_W/) 下载最新的 `.uf2` 文件。

### 2. 进入烧录模式

按住 Pico 2W 板上的 **BOOTSEL 按钮**不放，将 USB 数据线插入电脑。等待 2~3 秒后松开按钮，电脑上会出现一个名为 **RPI-RP2** 的可移动磁盘。

### 3. 拖拽烧录

将下载的 `.uf2` 固件文件直接拖拽到 `RPI-RP2` 磁盘中。文件复制完成后，磁盘会自动断开弹出，Pico 2W 会自动重启进入 MicroPython 模式——至此固件已烧录完成。

> **验证方法**：重新插拔 USB，通过后面的步骤进入 REPL 交互模式，看到 `>>>` 提示符即表示成功。

---

## 二、准备工作

### 安装 picocom（串口工具）

```sh
# Debian / Ubuntu
sudo apt install picocom

# macOS
brew install picocom

# Arch Linux
sudo pacman -S picocom
```

### 创建 Python 虚拟环境并安装 mpremote

mpremote 是 MicroPython 官方提供的命令行工具，用于与开发板通信、传输文件。

```sh
# 创建 Python 虚拟环境
python -m venv .env

# 激活虚拟环境（根据你的 shell 选择）
source .env/bin/activate         # bash / zsh
source .env/bin/activate.fish    # fish shell

# 安装 mpremote
pip install mpremote
```

### 查找串口设备

Pico 2W 连接后，在系统中会以一个串口设备的形式出现。用以下命令确认：

```sh
# Linux
ls /dev/ttyACM* /dev/ttyUSB*

# macOS
ls /dev/cu.usbmodem*
```

通常 Pico 2W 的设备路径是 `/dev/ttyACM0`。如果插了多个设备可能变成 `ttyACM1`，注意调整后续命令中的路径。

### 赋予串口权限

每次插入 Pico 都需要串口读写权限，有两种方式：

**方式一（临时）：手动赋权**
```sh
sudo chmod 666 /dev/ttyACM0
```
每次重新插拔后都需要再次执行。

**方式二（推荐）：添加 udev 规则**

创建 `/etc/udev/rules.d/99-pico.rules`：

```
SUBSYSTEM=="tty", ATTRS{idVendor}=="2e8a", MODE="0666"
```

然后重载 udev 规则：

```sh
sudo udevadm control --reload-rules
sudo udevadm trigger
```

之后每次插入 Pico 都会自动获得权限，一劳永逸。

> **替代方式**：将你的用户加入 `dialout` 组（`sudo usermod -aG dialout $USER`），注销重新登录后同样可以免 sudo 访问串口。

## 三、REPL 交互模式

REPL（Read-Eval-Print Loop）是 MicroPython 的交互式环境，可以直接输入 Python 命令并实时看到执行结果，非常适合调试和快速验证。

```sh
picocom -b 115200 /dev/ttyACM0
```

连接成功后可以看到 `>>>` 提示符，试试输入：

```python
print("Hello Pico 2W!")
machine.freq()      # 查看当前主频
import sys; sys.version  # 查看 MicroPython 版本
```

退出 picocom：按 `Ctrl + A`，再按 `Ctrl + X`。

## 四、上传脚本并运行

### 编写脚本

```sh
nvim main.py   # 也可以用你喜欢的任意编辑器
```

### 上传到 Pico 并重启运行

```sh
mpremote connect /dev/ttyACM0 cp main.py :main.py
mpremote connect /dev/ttyACM0 reset
```

或者简写为（自动连接）：

```sh
mpremote cp main.py :main.py
mpremote reset
```

### boot.py 与 main.py

Pico 上电后会自动按以下顺序执行脚本：

1. **`boot.py`** — 最先运行，适合做硬件初始化、Wi-Fi 连接等一次性配置
2. **`main.py`** — 在 boot.py 之后运行，放置主要业务逻辑

> 如果只需要一个文件，只写 `main.py` 即可。`boot.py` 不是必须的。

## 六、MicroPython 核心接口速查

| 功能分类   | 主要接口 / 模块                    | 核心功能                            | 备注                                            |
| :--------- | :--------------------------------- | :---------------------------------- | :---------------------------------------------- |
| 基本 GPIO  | `machine.Pin`                      | 控制 GPIO 引脚，读写高低电平        | 支持内部上拉/下拉电阻                           |
| 板载 LED   | `machine.Pin("LED", ...)`          | 控制 Pico 2W 板载 LED               | 由 WiFi 芯片控制，需用 `"LED"` 名称初始化       |
| PWM 输出   | `machine.PWM`                      | 生成 PWM 信号，调节舵机、LED 亮度等 | 舵机通常用 50Hz（20ms 周期）                    |
| 时间相关   | `utime` / `time`                   | 延时、时间戳等                      | 常用 `utime.sleep(秒)` / `utime.sleep_ms(毫秒)` |
| 蓝牙 (BLE) | `bluetooth`, `BLESimplePeripheral` | 作为 BLE 外设，与手机等设备通信     | 仅支持 BLE，不支持经典蓝牙                      |
| Wi-Fi      | `network`                          | 连接无线网络，实现物联网功能        | —                                               |

## 七、示例代码

### 1. 板载 LED 闪烁

```python
import machine
import utime

led = machine.Pin("LED", machine.Pin.OUT)

while True:
    led.toggle()
    utime.sleep(1)
```

Pico 2W 的板载 LED 与普通 GPIO 不同，它由 Wi-Fi 芯片（CYW43439）控制，必须使用 `"LED"` 这个名称来初始化。

### 2. 读取按键输入（GPIO 14）

```python
import machine
import utime

button = machine.Pin(14, machine.Pin.IN, machine.Pin.PULL_UP)

while True:
    if button.value() == 0:  # 按键按下时引脚接地，读到低电平
        print("按钮被按下！")
        utime.sleep(0.2)      # 简单防抖
```

### 3. Wi-Fi 连接

```python
import network
import utime

wlan = network.WLAN(network.STA_IF)
wlan.active(True)

SSID = "你的WiFi名称"
PASSWORD = "你的WiFi密码"

if not wlan.isconnected():
    print("正在连接 Wi-Fi...")
    wlan.connect(SSID, PASSWORD)

    timeout = 10
    while not wlan.isconnected() and timeout > 0:
        utime.sleep(1)
        timeout -= 1

if wlan.isconnected():
    print("Wi-Fi 已连接！")
    print("IP 地址:", wlan.ifconfig()[0])
else:
    print("Wi-Fi 连接失败")
```

### 4. 蓝牙（BLE）无线控制

> **注意**：本例使用 `BLESimplePeripheral` 辅助类，该类并非 MicroPython 内置模块。需要先从 [MicroPython 示例仓库](https://github.com/micropython/micropython/blob/master/examples/bluetooth/ble_simple_peripheral.py) 下载 `ble_simple_peripheral.py` 并上传到 Pico 的根目录（`mpremote cp ble_simple_peripheral.py :ble_simple_peripheral.py`）。

```python
from machine import Pin
import bluetooth
from ble_simple_peripheral import BLESimplePeripheral
import utime

ble = bluetooth.BLE()
sp = BLESimplePeripheral(ble, "pico2w")

red = Pin(13, Pin.OUT)
green = Pin(11, Pin.OUT)
yellow = Pin(12, Pin.OUT)

def on_rx(data):
    print("收到数据:", data)
    cmd = data.decode('utf-8').rstrip('\r\n')

    red.off()
    green.off()
    yellow.off()

    if cmd == "R" or cmd == "r":
        red.on()
    elif cmd == "G" or cmd == "g":
        green.on()
    elif cmd == "Y" or cmd == "y":
        yellow.on()

while True:
    if sp.is_connected():
        sp.on_write(on_rx)
    utime.sleep(0.1)
```

手机端可用 nRF Connect 或 LightBlue 等 BLE 调试 App 搜索 "pico2w" 设备并连接，发送 R / G / Y 控制对应颜色的 LED。

## 八、常见问题排错

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| `could not open port /dev/ttyACM0: Permission denied` | 没有串口权限 | 执行 `sudo chmod 666 /dev/ttyACM0`，或配置 udev 规则 |
| `No device found` / `no serial port available` | 设备路径不对 | 用 `ls /dev/ttyACM*` 确认实际路径，或检查 USB 是否松动 |
| `Device is busy` | 串口被其他程序占用 | 关闭正在使用的 picocom、Thonny 等，用 `ps aux | grep picocom` 检查 |
| 上传文件后没有运行 | `main.py` 未在根目录 | 用 `mpremote ls` 检查文件位置，确保文件名正确 |
| 按 BOOTSEL 没出现 RPI-RP2 磁盘 | 操作顺序不对 | 先按住 BOOTSEL **再**插入 USB，等待 2~3 秒再松手 |
| BLE 搜不到设备 | `ble_simple_peripheral.py` 缺失 | 确认已将该文件上传到 Pico 根目录 |

## 总结

Pico 2W 的 MicroPython 开发归结为三步：

1. **烧录固件** — 拖拽 UF2 文件
2. **连接与交互** — `picocom` 进入 REPL
3. **编写与上传** — `mpremote` 传输脚本

在此基础上，`machine` 模块负责硬件底层控制（GPIO、PWM 等），`network` 和 `bluetooth` 模块负责无线通信，将两者结合即可构建各种物联网交互设备。

> From zero to running — setup, connect, and control your Pico 2W

The Raspberry Pi Pico 2W is a microcontroller board powered by the RP2350 chip, with built-in Wi-Fi and BLE. It can be easily programmed with MicroPython. This guide walks you through everything from flashing firmware to building real projects.

---

## 1. Flashing MicroPython Firmware

A brand-new Pico 2W comes blank — you need to flash MicroPython firmware first.

### 1. Download the Firmware

Go to the [MicroPython download page](https://micropython.org/download/RPI_PICO2W/) and download the latest `.uf2` file.

### 2. Enter Bootloader Mode

Hold down the **BOOTSEL button** on the Pico 2W, then plug the USB cable into your computer. Wait 2–3 seconds before releasing the button. A removable drive named **RPI-RP2** should appear on your computer.

### 3. Drag and Drop

Drag the downloaded `.uf2` file onto the `RPI-RP2` drive. Once the copy finishes, the drive will automatically disconnect and eject. The Pico 2W will reboot into MicroPython mode — firmware flashing is complete.

> **Verify**: Unplug and re-plug the USB. Follow the REPL steps below — if you see the `>>>` prompt, you're all set.

---

## 2. Prerequisites

### Install picocom (Serial Terminal)

```sh
# Debian / Ubuntu
sudo apt install picocom

# macOS
brew install picocom

# Arch Linux
sudo pacman -S picocom
```

### Create a Python Virtual Environment & Install mpremote

mpremote is the official MicroPython CLI tool for communicating with and transferring files to your board.

```sh
# Create a virtual environment
python -m venv .env

# Activate it (choose based on your shell)
source .env/bin/activate         # bash / zsh
source .env/bin/activate.fish    # fish shell

# Install mpremote
pip install mpremote
```

### Find the Serial Port

When connected, the Pico 2W appears as a serial device. Use these commands to find it:

```sh
# Linux
ls /dev/ttyACM* /dev/ttyUSB*

# macOS
ls /dev/cu.usbmodem*
```

The typical device path is `/dev/ttyACM0`. If you have multiple devices connected, it may be `ttyACM1` — adjust the path accordingly.

### Grant Serial Port Permissions

You need read/write access to the serial port. There are two approaches:

**Option A (Temporary): Manual Permission**

```sh
sudo chmod 666 /dev/ttyACM0
```

You'll need to re-run this every time you unplug and reconnect the Pico.

**Option B (Recommended): udev Rule**

Create `/etc/udev/rules.d/99-pico.rules`:

```
SUBSYSTEM=="tty", ATTRS{idVendor}=="2e8a", MODE="0666"
```

Then reload udev rules:

```sh
sudo udevadm control --reload-rules
sudo udevadm trigger
```

After this, permissions are granted automatically whenever you plug in the Pico.

> **Alternative**: Add your user to the `dialout` group (`sudo usermod -aG dialout $USER`), then log out and back in.

## 3. REPL Interactive Mode

REPL (Read-Eval-Print Loop) is MicroPython's interactive environment. You can type Python commands and see results immediately — great for debugging and quick experiments.

```sh
picocom -b 115200 /dev/ttyACM0
```

Once connected, you'll see the `>>>` prompt. Try typing:

```python
print("Hello Pico 2W!")
machine.freq()
import sys; sys.version
```

To exit picocom: press `Ctrl + A`, then `Ctrl + X`.

## 4. Uploading and Running Scripts

### Write Your Script

```sh
nvim main.py   # or any editor you prefer
```

### Upload to Pico & Reboot

```sh
mpremote connect /dev/ttyACM0 cp main.py :main.py
mpremote connect /dev/ttyACM0 reset
```

Or in short form (auto-connect):

```sh
mpremote cp main.py :main.py
mpremote reset
```

### boot.py vs main.py

When the Pico powers on, it runs scripts in this order:

1. **`boot.py`** — Runs first. Good for hardware init, Wi-Fi setup, etc.
2. **`main.py`** — Runs after boot.py. Put your main application logic here.

> If you only need one file, just write `main.py`. `boot.py` is optional.

## 6. MicroPython Core API Reference

| Category    | Key Interface / Module            | Core Functionality                   | Notes                                       |
| :---------- | :-------------------------------- | :----------------------------------- | :------------------------------------------ |
| GPIO        | `machine.Pin`                     | Control GPIO pins, read/write levels | Supports internal pull-up/pull-down         |
| Onboard LED | `machine.Pin("LED", ...)`         | Control the Pico 2W onboard LED      | Controlled via Wi-Fi chip, use `"LED"` name |
| PWM         | `machine.PWM`                     | Generate PWM signals                 | Servos typically use 50Hz (20ms period)     |
| Time        | `utime` / `time`                  | Delays, timestamps                   | `utime.sleep(sec)` / `utime.sleep_ms(ms)`   |
| BLE         | `bluetooth`, `BLESimplePeripheral`| BLE peripheral, communicate with phone| BLE only, no classic Bluetooth             |
| Wi-Fi       | `network`                         | Connect to wireless networks         | —                                           |

## 7. Code Examples

### 1. Onboard LED Blink

```python
import machine
import utime

led = machine.Pin("LED", machine.Pin.OUT)

while True:
    led.toggle()
    utime.sleep(1)
```

The Pico 2W's onboard LED is controlled by the Wi-Fi chip (CYW43439), not a regular GPIO. You must use the name `"LED"` to initialize it.

### 2. Read Button Input (GPIO 14)

```python
import machine
import utime

button = machine.Pin(14, machine.Pin.IN, machine.Pin.PULL_UP)

while True:
    if button.value() == 0:  # button pressed — pin pulled low
        print("Button pressed!")
        utime.sleep(0.2)      # simple debounce
```

### 3. Wi-Fi Connection

```python
import network
import utime

wlan = network.WLAN(network.STA_IF)
wlan.active(True)

SSID = "YourWiFiName"
PASSWORD = "YourWiFiPassword"

if not wlan.isconnected():
    print("Connecting to Wi-Fi...")
    wlan.connect(SSID, PASSWORD)

    timeout = 10
    while not wlan.isconnected() and timeout > 0:
        utime.sleep(1)
        timeout -= 1

if wlan.isconnected():
    print("Wi-Fi connected!")
    print("IP address:", wlan.ifconfig()[0])
else:
    print("Wi-Fi connection failed")
```

### 4. BLE Wireless Control

> **Note**: This example uses the `BLESimplePeripheral` helper class, which is **not** a built-in MicroPython module. Download `ble_simple_peripheral.py` from the [MicroPython examples repo](https://github.com/micropython/micropython/blob/master/examples/bluetooth/ble_simple_peripheral.py) and upload it to the Pico's root directory (`mpremote cp ble_simple_peripheral.py :ble_simple_peripheral.py`).

```python
from machine import Pin
import bluetooth
from ble_simple_peripheral import BLESimplePeripheral
import utime

ble = bluetooth.BLE()
sp = BLESimplePeripheral(ble, "pico2w")

red = Pin(13, Pin.OUT)
green = Pin(11, Pin.OUT)
yellow = Pin(12, Pin.OUT)

def on_rx(data):
    print("Received:", data)
    cmd = data.decode('utf-8').rstrip('\r\n')

    red.off()
    green.off()
    yellow.off()

    if cmd == "R" or cmd == "r":
        red.on()
    elif cmd == "G" or cmd == "g":
        green.on()
    elif cmd == "Y" or cmd == "y":
        yellow.on()

while True:
    if sp.is_connected():
        sp.on_write(on_rx)
    utime.sleep(0.1)
```

Use a BLE debugging app (nRF Connect, LightBlue, etc.) on your phone to scan for "pico2w", connect, and send R / G / Y to control each LED color.

## 8. Troubleshooting

| Problem | Likely Cause | Solution |
|---------|-------------|----------|
| `could not open port /dev/ttyACM0: Permission denied` | No serial permission | Run `sudo chmod 666 /dev/ttyACM0` or set up a udev rule |
| `No device found` / `no serial port available` | Wrong device path | Run `ls /dev/ttyACM*` to find the correct path; check USB connection |
| `Device is busy` | Port in use by another program | Close picocom, Thonny, etc. Check with `ps aux \| grep picocom` |
| File uploaded but not running | `main.py` not in root directory | Use `mpremote ls` to check; make sure the filename is correct |
| BOOTSEL doesn't show RPI-RP2 drive | Wrong button sequence | Hold BOOTSEL **before** plugging in USB, wait 2–3 seconds, then release |
| BLE device not found | `ble_simple_peripheral.py` missing | Make sure the file is uploaded to the Pico root |

Pico 2W MicroPython development in three steps:
